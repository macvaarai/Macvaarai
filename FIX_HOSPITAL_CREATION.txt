HOSPITAL CREATION FIX - 2026-06-02
===================================

PROBLEM:
--------
Error: "Error saving hospital: Failed to fetch"
Backend Error: 422 Unprocessable Entity (Validation Error)

ROOT CAUSE:
-----------
The backend endpoint was expecting JSON body with Pydantic model,
but frontend was sending FormData with form fields.
This mismatch caused FastAPI validation to fail with 422 error.

SOLUTION APPLIED:
-----------------
Updated backend endpoint (/admin/hospitals) to accept all parameters
as Form() instead of JSON Pydantic model.

Changes Made:
1. Changed create_hospital endpoint from JSON to FormData
2. Updated all field declarations to use Form(...)
3. Added JSON parsing for subscribed_models field
4. Updated update_hospital endpoint similarly

--------------

   - Line 703-749: update_hospital endpoint
