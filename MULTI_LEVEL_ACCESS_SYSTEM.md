### **NEW TABLE 2: Organization Admins**

```sql
CREATE TABLE organization_admins (
    id INT PRIMARY KEY AUTO_INCREMENT,
    organization_id INT NOT NULL,
    user_id INT NOT NULL,
    name VARCHAR(255),
    email VARCHAR(255) UNIQUE,
    phone VARCHAR(20),
    role VARCHAR(100),  -- org_admin, org_manager, org_support
    access_level VARCHAR(50),  -- full, limited
    created_at TIMESTAMP,
    last_login TIMESTAMP,
    is_active BOOLEAN DEFAULT TRUE,
    
    FOREIGN KEY (organization_id) REFERENCES organizations(id),
    FOREIGN KEY (user_id) REFERENCES users(id)
);

EXAMPLE DATA:
в”ңв”Җ Admin 1: Vijay Care

в”Ӯ  в”ңв”Җ Role: org_admin

   в”ңв”Җ Name: Modi Care Manager
   в”ңв”Җ Organization: Modi Care
   в”ңв”Җ Role: org_manager
   в””в”Җ Access: Limited
```

### **UPDATE EXISTING TABLE: Hospitals**

```sql
ALTER TABLE hospitals ADD COLUMN organization_id INT;
ALTER TABLE hospitals ADD FOREIGN KEY (organization_id) REFERENCES organizations(id);

EXAMPLE:
Hospital 1: Apollo Hospital
в”ңв”Җ Organization: Vijay Care
в”ңв”Җ Hospital ID: H-001
в””в”Җ Org ID: 1 (Vijay Care)

Hospital 2: Max Hospital
в”ңв”Җ Organization: BJP Care
в”ңв”Җ Hospital ID: H-002
в””в”Җ Org ID: 2 (BJP Care)
```

### **UPDATE EXISTING TABLE: Users**

```sql
ALTER TABLE users ADD COLUMN organization_id INT;
ALTER TABLE users ADD COLUMN role_type VARCHAR(100);
-- role_type: super_admin, org_admin, hospital_admin, doctor, patient, staff

EXAMPLE ROLES:
в”ңв”Җ Super Admin: Can see all organizations & hospitals
в”ңв”Җ Org Admin: Can see only their organization's hospitals
в”ңв”Җ Hospital Admin: Can see only their hospital
в”ңв”Җ Doctor: Can see only their hospital's patients
в””в”Җ Patient: Can see only themselves
```
